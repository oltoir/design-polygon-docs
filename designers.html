🎨 Для дизайнеров

Работаем в Polygon Launcher

Установить инструменты, открыть полигон в лаунчере, собрать первый компонент через Claude и передать разработчику. Базовые навыки терминала не нужны.

Что ты получишь после этой инструкции

🛠 Установленные инструменты

Claude Code и Polygon Launcher на твоей машине. Один раз — и забыл.

🖥 Графический интерфейс

Терминал не нужен. Всё в одном окне: чат, превью, git, dev-сервер.

⏱ Скорость 30–60 минут

Собрать продакшн-готовый widget и передать разработчику.

📏 Понимание границ

Что ИИ делает за тебя, а что остаётся работой фронта.

📘

Если ты раньше не открывал терминал — ничего страшного. Шаги ниже расписаны так, что нужно только копировать команды и нажимать Enter.

Зачем дизайнеру писать код в 2026

Раньше дизайнер делал макет в Figma, фронт переписывал его в код. На карточку уходило 4–8 часов плюс несколько раундов синхронизации.

Сейчас всё иначе. Claude при правильно подготовленном контексте стабильно собирает верстку по правилам команды. Дизайн-полигон — это и есть подготовленный контекст.

Что это значит для тебя

💬 Диалог на русском

Продакшн-готовый код через разговор. Никакого «учить программирование» — нужно только чётко описывать визуальные требования.

👁 Живое превью

Видишь результат в браузере и итерируешь. Без ожидания фронта.

🔄 Один артефакт

Перестаёшь синхронизировать Figma и код. Артефакт один.

⚡ Разгрузка фронта

Фронт занимается логикой и перформансом. Каждый делает то, в чём силён.

💪

Это не «дизайнер заменяет фронта». Это новый формат работы, в котором каждый занимается своей сильной стороной: ты — визуальной частью, фронт — инженерной.

Что нужно установить (один раз, ~30 минут)

1

Node.js (через nvm)

Node — это среда, в которой запускаются современные веб-проекты.

macOS / Linux

Открой Terminal (на Mac: ⌘+Space → «Terminal» → Enter):

bashустановка nvm
curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.7/install.sh | bash

Закрой Terminal и открой заново — иначе nvm не «подхватится».

bashустановка Node LTS
nvm install --lts
nvm use --lts
node -v

Должен показать что-то вроде v22.12.0.

Windows

  1. Зайди на nodejs.org
  2. Скачай LTS-версию (зелёная плашка «Recommended for Most Users»)
  3. Запусти установщик, нажимай Next до конца
  4. В PowerShell проверь: node -v
2

Git

Git нужен для версионирования — это «история изменений» проекта.

OSКоманда
macOS xcode-select --install
Ubuntu / Debian sudo apt update && sudo apt install -y git
Windows git-scm.com/download/win → установщик с настройками по умолчанию

Один раз настрой автора коммитов:

bashgit config
git config --global user.name "Имя Фамилия"
git config --global user.email "your.email@aviata.me"
3

Claude Code CLI

Это сам Claude. Лаунчер использует его «под капотом», но для первичной авторизации нужно запустить вручную один раз.

bashустановка Claude Code
npm install -g @anthropic-ai/claude-code
claude --version

Авторизация:

bashавторизация
claude
# в открывшемся интерфейсе набери:
/login

Откроется браузер с окном входа Anthropic — войди по рабочей Max-подписке. После входа выйди из Claude (Ctrl+C дважды).

4

Polygon Launcher

Главный инструмент. Скачай последний релиз:

5

Клонировать полигон

Получи у разработчика URL репозитория. Затем в терминале:

bashклонирование
mkdir -p ~/Desktop/polygons
cd ~/Desktop/polygons
git clone https://github.com/aviata/booking-design-polygon.git

Через минуту папка ~/Desktop/polygons/booking-design-polygon/ появится на компьютере. Запомни этот путь — нужен на следующем шаге.

Первый запуск Polygon Launcher

Открой Polygon Launcher. Первое окно встретит тебя списком полигонов слева.

Главное окно

Главное окно: Sidebar слева, Preview по центру, Чат/Git/Обзор/Логи справа

Три зоны лаунчера

📋 Sidebar (слева)

Список полигонов, кнопки «Add Polygon» и «Reload». Иконки рядом показывают статус окружения: зелёная — ок, жёлтая — warning, красная — ошибка.

🖥 Preview (центр)

Браузер с dev-сервером. Сверху — переключатели устройств: Mobile / Tablet / Desktop / Fluid (22 пресета девайсов).

💬 Правая панель

Четыре вкладки: Чат (с Claude), Git (ветки, коммиты), Логи (dev-сервер), Обзор (статус окружения).

Что видно на скриншоте

  • Slabar — Sidebar. Два зарегистрированных полигона: «Aviax Design Polygon» (npm, Node 18.18.2) и «Freedom Miniapp Design Polygon» (pnpm, Node ≥22). Зелёные индикаторы «Готов к запуску».
  • Центр. Открыт Aviax. Видна строка «Vue 2.7 / npm — конкурс продакшн-стек Aviata», кнопка Запустить (синяя, справа сверху), переключатели Mobile (375) / Tablet (768) / Desktop (1280) / Fluid.
  • Подсказка в центре. «Полигон не запущен. Нажми Запустить сверху» — это состояние до первого запуска dev-сервера.
  • Чат справа. Открыт по умолчанию. Видна подсказка: «Перетащи скриншот в окно или начни вводом. Все плагины, .claude/-настройки и слэш-команды подгружаются автоматически.»
  • Statusbar внизу. Слева — селектор модели «auto». Справа — индикатор «Backend OK» и иконка уведомлений.

Выбор модели и effort

Внизу окна — статусбар с текущей моделью и уровнем effort.

МодельКогда использоватьСкорость / стоимость
Sonnet 4.6 Основная рабочая лошадка. 95% задач. баланс
Opus 4.7 Сложные компоненты, нестандартные решения. дороже, медленнее
Haiku 4.5 Мелкие правки (цвет, отступ, переименовать). быстро, дёшево

Effort — насколько глубоко Claude «думает»:

low / medium

Для типовых задач. Дефолт.

high / xhigh

Когда нужно подумать дольше (сложная архитектура).

max

Экстремальные случаи. Долго и дорого.

💡

Дефолт Sonnet 4.6 + medium покрывает почти всё. Переключайся на другое только если есть конкретная причина.

Первый компонент за 30 минут

Сценарий — booking-app, нужна карточка отеля для списка результатов поиска.

1

Создать ветку

Открой Git-панель. Нажми Create New Branch. Введи имя: feature/hotel-card. Нажми Create.

2

Запустить slash-команду

Открой Чат. Набери / — появится список команд. Выбери /make-template.

3

Фаза 1 — уточнение

Claude задаст вопросы (по очереди или списком). Отвечай развёрнуто:

Вопрос: Что за компонент?
Ответ: Карточка отеля для списка результатов поиска.

Вопрос: Где он будет жить (FSD-слой и путь)?
Ответ: widget, путь src/widgets/hotel-card/.

Вопрос: Какие данные показывает?
Ответ:

  • imageUrl: string — главное фото
  • name: string — название отеля
  • stars: number — звёзды (1–5)
  • rating: number — рейтинг (0.0–10.0)
  • reviewsCount: number — отзывов
  • pricePerNight: number — цена в тенге
  • city: string — город

Вопрос: Состояния?
Ответ: Default, hover (приподнимается, тень мягче), focus (синяя обводка), selected (синий фон, белый текст).

Вопрос: Адаптивность?
Ответ: Mobile + Desktop. На мобиле — фото сверху; на десктопе — фото слева.

После всех ответов Claude выдаст summary. Если всё верно — отвечай: «да, делай». Если правки — «всё ок, но добавь ещё …».

4

Фаза 2 — реализация

Появятся permission-диалоги. Нажимай Always для операций создания файлов, Allow once для bash-команд.

Через 1–3 минуты появятся файлы:

src/widgets/hotel-card/ ├── HotelCard.tsx ├── types.ts └── index.ts
5

Смотри в Preview, итерируй

Переключи устройство в Preview: посмотри на iPhone, потом на Desktop.

Пиши обычные правки в чат:

  • «Увеличь padding снизу до 16px»
  • «Сделай тень мягче»
  • «Цена должна быть жирнее и крупнее»
  • «На мобиле название отеля обрезается — добавь truncate в 2 строки»
🛡

Если Claude использовал запрещённый цвет (например text-gray-500) или произвольный spacing (p-[13px]) — сработает hook, операция блокируется, Claude сам переиграет с правильным токеном. Тебе ничего делать не нужно.

6

Commit + Push

Когда устраивает — открой Git-панель:

  • В Status — все изменённые файлы.
  • В поле Commit message: feat: add HotelCard widget.
  • Нажми Commit, потом Push.
7

Open Pull Request

В GitHub-интерфейсе → New Pull Request. В описании:

mdописание PR
## Что сделано
Новый виджет HotelCard — карточка отеля для списка результатов поиска.

## Скриншоты
- Mobile (iPhone 15): [перетяни скриншот Preview]
- Desktop: [перетяни скриншот Preview]

## Ветка
feature/hotel-card

Назначь reviewer'ом разработчика проекта (узнай у тимлида).

Чек-лист перед запросом ревью

Прежде чем нажать «Request Review» в PR — пройдись по списку:

  • Компонент выглядит как на Figma-макете (или есть согласованное отклонение)
  • Превью открывается без ошибок (нет белого экрана)
  • Logs-панель не показывает красных строк
  • Все интерактивные состояния проверены: hover, focus, disabled, loading
  • Адаптивность: переключи Mobile / Tablet / Desktop — нигде не «уезжает» за края, текст не обрезается
  • Закоммитил с осмысленным сообщением (не update, а feat: add HotelCard widget)
  • Запушил в feature-ветку, не в main!
  • Открыл PR с описанием и приложенными скриншотами (Mobile + Desktop)

Если все галочки — нажимай Request Review. Разработчик пройдётся со своим (более длинным) чек-листом и подскажет, если что-то не так.

Что ИИ делает за тебя, а что — нет

✓ Claude делает

  • Пишет JSX / Vue-разметку по описанию или скриншоту
  • Подбирает токены из палитры проекта
  • Расставляет отступы по 4px-grid
  • Создаёт структуру файлов (компонент + index.ts + types.ts)
  • Подсказывает имена пропсов, переменных, файлов
  • Рефакторит код по запросу
  • Объясняет любую строчку кода на русском
  • Делает коммиты, пушит, создаёт ветки
  • Воспроизводит макет с Figma-скриншота

✕ Claude НЕ делает (работа фронта)

  • Подключает API и реальные данные
  • Подключает store (Pinia / Zustand / Redux)
  • Подключает router и навигацию
  • Подключает i18n / переводы
  • Подключает аналитику
  • Делает unit / integration / E2E тесты
  • Решает архитектурные вопросы вне полигона
  • Деплоит в production
  • Оптимизирует перформанс (бандл, hydration)
📏

Правило простое: задача про визуал и интеракции — твоя зона + Claude. Задача про данные, логику, связи между частями — зона фронта.

Полезные приёмы

🖼 Скрин Figma → Чат

Скриншот фрейма (⌘+Shift+4 на Mac), перетащи в поле чата. Claude увидит макет.

🔁 Маленькие итерации

«Сделай карточку → теперь увеличь тень → теперь добавь рейтинг» — лучше, чем одно огромное описание.

🗣 Не бойся переспрашивать

«Нет, я имел в виду другое, объясни ещё раз» — это нормально.

Если что-то сломалось

🔴 Превью не открывается / белый экран

  1. Открой вкладку Логи
  2. Найди красные строки (ERR)
  3. Скопируй их и отправь в Чат: «Вот ошибка из dev-сервера: …. Помоги починить»
  4. Claude посмотрит и предложит фикс

🟡 Claude не отвечает / ошибка авторизации

  1. Открой вкладку Обзор
  2. Проверь строку Claude auth — должна быть зелёная галочка
  3. Если красный крестик — открой терминал, набери claude /login
  4. Вернись в лаунчер, нажми «Reload»

🟢 Hook ругается на text-gray-500 или p-[13px]

Это так и должно быть. Hook защищает проект от несогласованных стилей. Claude сам перепишет на правильный токен — просто подожди.

Если очень-очень нужно исключение — добавь в первую строку файла // claude-allow:tokens. Но лучше согласовать с разработчиком.

⚪️ Polygon Launcher не запускается

  • macOS: ПКМ на иконке → «Открыть» (обход Gatekeeper)
  • Linux: проверь sudo apt install libfuse2
  • Windows: запусти от имени администратора, проверь антивирус

Совсем непонятная ошибка → Telegram @oltoir. Скриншот ошибки + что делал в момент сбоя.
